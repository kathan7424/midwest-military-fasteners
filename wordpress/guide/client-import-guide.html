1 Quick Start

The 4-step version:
  1. Download import-template.csv from WooCommerce → Parts Import and fill it in
  2. Upload photos, spec sheets, and certificates to their folders (see Section 5)
  3. Click Preview (no changes) — review the report for warnings
  4. Click Import CSV when the preview looks good
✓ Do
  • Keep columns in the exact same order as the template
  • Save as CSV (comma-separated), not XLSX
  • Add a category context row before each new group of products
  • Use the P/N (Part Number) as your filenames for auto-matching
  • Run Preview before every real import
✕ Don't
  • Don't add, remove, or reorder columns
  • Don't leave the P/N column empty on a product row
  • Don't use the same P/N (SKU) more than once in the file
  • Don't import without previewing first
  • Don't include column headers from a second copy of the file

4 Pricing Tiers

The website shows tiered pricing: the customer sees a better price per package the more they order. The website handles the logic automatically — you just fill in the prices.

ColumnTierCustomer sees this price when ordering…Also sets
10 — 1 PKG COSTTier 11–2 packagesWC regular price (the price shown on the website)
11 — 3 PKG COSTTier 23–4 packages—
14 — 5 PKG COSTTier 35–9 packages—
17 — 10 PKG COSTTier 410+ packages—
At minimum, fill in column 10 (1 PKG COST). The other tiers are optional — leave them blank if you don't offer volume discounts. Prices can include a dollar sign ($) — it is removed automatically.
Import warning: A product imported with no price in column 10 will appear on the website with a $0.00 price. The Preview mode will warn you before this happens.

5 Photos, Spec Sheets & Certificates

Step 1 — Upload your files to the right folders

Before importing the CSV, upload your files to these folders in the WordPress Media Library (ask your developer to set up SFTP access or use the Media → Add New screen):

File TypeFolderColumnAuto-name pattern
Product photos (JPG/PNG/WEBP)product-images/34 — PRODUCT IMAGE{P/N}.jpg
Spec sheet PDFsproduct-specs/35 — SPEC SHEET{P/N}-spec.pdf
Certificate PDFs or imagesproduct-certs/36 — CERTIFICATE{P/N}-cert.pdf

Auto-matching (easiest approach)

If you name your files after the part number, you can leave columns 34–36 completely blank in the CSV — the importer finds them automatically:

Example: For part number MS35307-303:
  • Photo: name it MS35307-303.jpg → upload to product-images/
  • Spec sheet: name it MS35307-303-spec.pdf → upload to product-specs/
  • Certificate: name it MS35307-303-cert.pdf → upload to product-certs/
Leave columns 34, 35, 36 blank. The system matches them automatically.

Multiple product photos

To add a gallery (more than one photo), list filenames separated by commas in column 34:

MS35307-303.jpg, MS35307-303-side.jpg, MS35307-303-angle.jpg

The first image becomes the main product photo. The rest become the gallery.

Category thumbnail photos

Column 1 (CATEGORY IMAGE LOCATION) sets the thumbnail shown on the category/sidebar in the website. You only need to set it once on the category context row. Use a filename from the product-images/ folder or a full URL.

7 Preview First, Then Import

1
Go to WooCommerce → Parts Import In your WordPress admin menu.
2
Upload your CSV file Click "Choose File" and select your completed spreadsheet.
3
Click "Preview (no changes)" You'll see a row-by-row report:
  • CREATE — new product will be added
  • UPDATE — existing product will be updated (only if "Update existing" is checked)
  • SKIP — product exists, will be left as-is
  • WARN — something to review (missing price, photo not found, etc.)
4
Fix any warnings, then click "Import CSV" Preview is always 100% safe — it never changes anything. Run it as many times as you need.
Preview is always safe. No products are created or changed during a preview run. Use it as many times as you like before committing to the real import.

8 Updating Products Later

To change prices, descriptions, stock, or any other field:

  1. Edit the same CSV file with the updated values
  2. Tick "Update existing products" on the import screen
  3. Preview first, then import

Products are matched by their P/N (SKU) — existing products are updated and new P/Ns are added. Products not in the file are left untouched.

Only the fields you fill in are updated. If you leave a column blank for an existing product, the current value in the database is kept.
