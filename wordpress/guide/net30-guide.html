What is Net 30?

Net 30 lets an approved business customer place an order without paying at checkout. A printed invoice ships with the goods and payment is due within 30 days.

No Payment at Checkout

No credit card, no Stripe charge. The customer places the order and receives an invoice with their shipment.

Admin-Approved Only

Only customers explicitly enabled by admin can see or use this option. Guests are always excluded — login required.

Manual Fulfilment

Admin ships the order via Shippo, collects payment outside the platform, then marks the order Completed.

ℹ️

Under the hood: Net 30 uses WooCommerce's built-in Cash on Delivery (COD) gateway — renamed "Net 30 — Purchase Order Terms" in the WC settings. COD is the WooCommerce standard approach for deferred payment.

⚠️

Why "Processing" and not "On Hold"? Shippo's "Create Label" button only appears on Processing orders. A custom hook forces Net 30 orders out of WooCommerce's default COD status (On Hold) into Processing on creation. If you ever see a Net 30 order stuck on On Hold, manually change it to Processing.

WooCommerce Settings

The COD gateway must be correctly configured before Net 30 works. Do this once.

  1. Go to WP Admin → WooCommerce → Settings → Payments Find the "Cash on Delivery" row in the payment methods list.
  2. Click "Manage" on Cash on Delivery Opens the COD gateway settings page.
  3. Fill in these exact fields and save See the table below for values.

Enabling / Disabling Customers

Two methods — both update the same flag instantly. Use whichever is faster for your workflow.

Method A — Users List Fastest
Method B — User Profile
  1. WP Admin → Users Opens the full users list. The "Net 30" column appears on the right side of the table.
  2. Find the customer Use the search box at the top if needed.
  3. Look at the Net 30 column Disabled grey button = customer cannot use Net 30
    ✓ Eligible green button = customer can use Net 30
  4. Click the button to toggle Saves instantly via AJAX — no page reload. Button colour changes immediately to confirm.
  1. WP Admin → Users → click the customer's name Opens the full user profile edit page.
  2. Scroll down to "Net 30 Payment Terms" section Located below the standard WooCommerce billing fields.
  3. Check or uncheck "Net 30 eligible" Checked = customer can use Net 30. Unchecked = standard credit card only.
  4. Click "Update User" to save Takes effect immediately — customer will see the Net 30 option on their next checkout page load.
✅

Only shop managers and administrators can manage Net 30 eligibility. The option is not visible to other user roles.

Fulfilling a Net 30 Order

Day-to-day workflow once an order lands in WP Admin:

  1. Order arrives in WP Admin → Orders as "Processing" You'll see the payment method listed as "Cash on Delivery" (WC's internal label for Net 30). The customer's email and addresses are pre-filled.
  2. Review the order details Verify billing address, shipping address, items, and quantities. Add any internal notes if needed.
  3. Create the shipping label via Shippo Scroll to the Shippo section within the order. Select carrier, service level, and package dimensions → click Generate Label. The PDF label downloads automatically.
  4. Print and attach the invoice Use WooCommerce's built-in invoice (WC PDF Invoices plugin if installed, or the print button on the order detail page). Include it with the shipment as the 30-day payment notice.
  5. Ship the goods Attach the label, include the invoice, and hand off to the carrier. The tracking number will auto-sync back to the WooCommerce order via Shippo.
  6. Wait for payment (up to 30 days) The order stays as Processing while awaiting payment. Follow up manually by phone or email if needed — there is no automated reminder in WooCommerce.
  7. Mark "Completed" when payment is received WP Admin → open order → Order Status → Completed → Update. This closes the order and sends the WooCommerce completion email to the customer.
🚫

Don't change the status to "On Hold" while shipping — Shippo's label creation button only appears on Processing orders. Leave it as Processing until payment is received, then move to Completed.

Troubleshooting

Customer says they can't see "Net 30" at checkout

  • Confirm the customer is logged in — guests are always excluded
  • Check WP Admin → Users → Net 30 column — must show green ✓ Eligible
  • If grey: click to enable, then ask customer to hard-reload the checkout page
  • Check WC Settings → Payments: Cash on Delivery must be Enabled

Shippo "Create Label" button is missing on the order

  • Check the order status — it must be Processing
  • If status is On Hold: change it to Processing and save the order
  • Refresh the order page after changing the status

Net 30 option appears for everyone (not gated)

  • Check wordpress/functions.php for: require_once ... '/inc/net30.php'
  • Verify the woocommerce_available_payment_gateways filter is registered at priority 20
  • Check that non-eligible users have no mmf_net30_eligible user meta set

COD gateway not appearing in WC Payments list

  • Go to WC → Settings → Payments and enable "Cash on Delivery"
  • The "Net 30" Users column will not appear if net30.php failed to load
  • Check the PHP error log for any fatal errors in net30.php
