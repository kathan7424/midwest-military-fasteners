1
FR-SHIP-04 · CLIENT ACTION

Connect carriers in the Shippo dashboard

In the client's own Shippo account, connect the UPS and FedEx carrier accounts that will actually ship the orders.

  • Carrier and service costs bill to the client's account — that's their responsibility, not something this build handles.
  • This step happens entirely inside Shippo's dashboard. No code, no plugin yet.
2
WP ADMIN

Install the Shippo for WooCommerce plugin

Install and activate the plugin on WordPress, then paste in the client's Shippo API token.

Token handling rule The API token goes in the plugin's own WooCommerce settings row only — never in code, never committed to git, never pasted into a guide or a chat log.
3
WP ADMIN

Set the ship-from address

Enter MMF's warehouse address in the plugin (or WooCommerce → Settings → General → Store Address, which the plugin reads from). Every rate quote and every pre-filled Shippo Shipment uses this as the origin.

4
WOOCOMMERCE → SETTINGS → SHIPPING

Turn on Shippo rates for the US zone

Add the plugin's live-rate method to the US shipping zone. The checkout page needs zero changes — it already renders whatever shipping_rates[] the Store API returns.

Decide whether to keep the interim methods:

  • Free Shipping (Michigan zone) and Flat Rate (US) can stay active alongside Shippo — WooCommerce just lists every method as its own radio option.
  • Or remove them once live rates are confirmed working, if only carrier rates should show. Business call, not a technical one.
5
ALREADY BUILT — VERIFY ONLY

Confirm every product has weight + dimensions

Shippo cannot quote a rate — or auto-fill a shipment's Package section — without both. The CSV import already writes these to WooCommerce's native product fields and warns on anything missing.

Bonus automation already live The moment an order reaches "Processing," shippo-sync.php pre-creates the Shippo Shipment with parcel weight/dimensions pulled from the order — so the admin never sees "Package info required" in the Shippo dashboard.
6
WOOCOMMERCE → SETTINGS → INTEGRATION

Wire up the delivery webhook

This closes the loop: Shippo reports a package delivered → the WooCommerce order auto-completes → certificate emails and My Account document unlocks fire automatically.

  • Open WooCommerce → Settings → Integration → Shippo Webhook and copy the endpoint URL (/wp-json/custom/v1/shippo/webhook).
  • In the Shippo dashboard: Settings → API → Webhooks → Add Webhook → event track_updated → paste the URL.
  • Copy Shippo's signing secret back into the WooCommerce settings panel. Every incoming call is HMAC-verified — never leave this blank in production.
  • Leave Auto-Complete on Delivery on (default).

Turn on Debug log on the same settings page while testing — every webhook call (received / skipped / rejected / completed) lands in WooCommerce → Status → Logs under source mmf-shippo-webhook. Switch it off once verified.
