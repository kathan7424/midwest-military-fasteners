1. Import Flow

Upload CSVImport tab · nonce + manage_woocommerce verified
Parse rowsPositional (by column #, not header) — headers repeat in this format
Context / ProductBlank P/N = context row; else create/update product by SKU
Sideload mediaImages, spec, cert from uploads folders
SaveWC product + taxonomies + tiers + meta
Positional parsing: the file has multiple SHIPPING DIMENSIONS / SHIPPING WEIGHT headers, so the importer reads by column position. Never reorder or delete columns — keep the exact 36-column layout of the template.

Update existing: checkbox on the Import tab. When checked, a row whose P/N matches an existing SKU updates that product in place instead of being skipped.

2. Context Rows — Category & Series Cascade

A row with a blank P/N (column 5) doesn't create a product — it sets the active category + sub-category + series. Every product row after it inherits that context until the next context row.

CATEGORY IMAGE,PRODUCT CATEGORY,PRODUCT SUB CATEGORY,PART SERIES,P/N,DESCRIPTION,...
,SCREWS,HEX CAP SCREWS,MS35307,,,...                       ← context row (P/N blank)
,,,,MS35307-303,"1/4-20 X 1/2 HEX CAP SCREW ...",50,...    ← product → SCREWS > HEX CAP SCREWS > MS35307
,,,,MS35307-306,"1/4-20 X 3/4 HEX CAP SCREW ...",50,...    ← product (same context)
,SCREWS,HEX CAP SCREWS,MS35308,,,...                       ← new context — series changes to MS35308
,,,,MS35308-303,"1/4-28 X 1/2 HEX CAP SCREW ...",50,...

4. Media — Images, Gallery, Spec & Certificate

Folders (Settings tab)

ContentDefault folder
Imagesuploads/product-images/
Spec sheetsuploads/product-specs/
Certificatesuploads/product-certs/
Security: configured folders must resolve inside wp-content/uploads/ — anything else is rejected. Filenames are reduced to their basename (no ../ path tricks).

Resolution order (each file)

  1. Full URL in the cell → stored directly
  2. Filename → looked up in Media Library (reused if already imported)
  3. Not in library → sideloaded from the folder
  4. Cell blank → SKU fallback tried: {SKU}.jpg, {SKU}-spec.pdf, {SKU}-cert.pdf (also UPPERCASE)

Gallery example (column 34)

MS35307-303.jpg, MS35307-303-side.jpg, MS35307-303-top.jpg

First file = featured image, remaining = WC product gallery.

Manual editing after import: Product data → General tab has "Spec Sheet (PDF)" and "Certificate (PDF)" fields with an Upload / Select button (WP Media Library). Pricing tiers: "Spec Parts: Package Pricing" box. Shipping weight/dims: WC Shipping tab (Shippo reads these).

7. Troubleshooting

SymptomCause → Fix
"Download sample CSV" → template not foundimport-template.csv missing from theme root on the server → upload it
Imported, but sidebar/frontend unchangedPantheon Varnish (5-min TTL) or frontend cache → Dashboard → Clear Caches, wait ≤5 min
Images not attachedFilename mismatch (case-sensitive) or file missing from configured uploads folder
Series appears in two taxonomiesLegacy imports used product_series; live uses product-series. All code reads both — harmless
Duplicate categories (two "FLAT WASHERS")Same-name terms from imports. APIs merge them; product queries match all duplicates
Row skipped silentlyP/N blank (treated as context row) or duplicate SKU without "Update existing" checked
