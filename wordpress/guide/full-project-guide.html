Midwest Military Fasteners is a headless WooCommerce B2B storefront. The browser only talks to Next.js. WordPress/WooCommerce is the single source of truth for products, pricing, tax, stock, orders, and every store setting. No prices, currency symbols, or checkout behaviours are hardcoded — all come from WC admin.

Headless Checkout

WC Store API — address → shipping rates → TaxJar tax → Stripe UPE → 3DS → success page.

Tax Exemption

Upload cert on register or via My Account. Admin HMAC-email approval. Approved = $0 tax at checkout. Daily expiry cron.

Net 30 Terms

COD gateway repurposed. Per-customer admin flag. Checkout skips card entry. Gateway filtered server-side — never exposed to unlisted customers.

Order Documents

Spec sheets available after purchase. Certificates unlocked after order ships/completes. Customer notified by email. Bulk download.

Product Catalog

Spec-parts CSV import pipeline. Package quantity pricing tiers. Series/category filter sidebar. Tier pricing shown to logged-in users only.

My Account (Figma)

Sidebar layout: Orders, Certifications, Spec Sheets, Documents (Tax Cert), Addresses, Account Details, Logout.

Architecture

Browser (React + Stripe Elements iframes) │ HTTPS only — WP session cookies are HttpOnly/Secure, never readable by JS ▼ Next.js App Router (RSC pages · ISR · /api/* proxy · security headers) │ forwards WP auth cookies + X-MMF-Proxy shared secret header ▼ WordPress / WooCommerce (Pantheon, PHP 8.x) ├── wc/store/v1 → cart · checkout · coupons · shipping · payment ├── custom/v1 → auth · tax-exemption · orders · settings · search ├── spec-parts/v1 → catalog API (categories, products, series) ├── Stripe Gateway (UPE deferred intent — card charged server-side at WC checkout) ├── TaxJar → rooftop-accurate sales tax (needs full address) └── Shippo → labels + tracking (pending client credentials)
Cache Strategy
  • Static/ISR — menus, site settings, catalog pages: 5-min ISR (revalidate:300)
  • Dynamic / no-store — cart, auth, orders, account data: cache:"no-store"
  • React cache() — fetch_store_settings deduplicated within a request
  • Dev cache — in-memory 2-min TTL for static fetches (bypasses persistent Data Cache during dev)
Auth Flow
  • Login → WP sets wordpress_logged_in_* cookie
  • Next.js proxy forwards cookie + X-MMF-Proxy secret header
  • WP mmf_headless_cookie_auth validates cookie → sets current user
  • WC Store API reads session via Nonce + Cart-Token headers
  • Tax-exempt flag checked server-side per request

Authentication — Login & Register

Login
  • POST /api/auth/login → WP sets session cookie
  • Cookie forwarded via Next.js proxy on all subsequent requests
  • Redirect to /my-account on success
  • Open-redirect blocked — only internal paths allowed
Register
  • Fields: First name, Last name, Company (optional), Email, Password × 2
  • Optional: Sales tax exemption certificate upload + expiry date
  • Expiry date DatePicker: today + future selectable, only past dates disabled (minValue = today)
  • Zod validation: today or future date required if cert uploaded
  • On success → auto-login → /my-account
Password Reset (Headless Flow)
  • User submits email on /forgot-password → POST /api/auth/forgot-password
  • WP: get_password_reset_key() generates a secure key, custom wp_mail() sends email with link to {MMF_FRONTEND_URL}/reset-password?key=...&login=...
  • MMF_FRONTEND_URL constant (or mmf_frontend_url filter) in wp-config.php sets the Next.js domain
  • User clicks link → Next.js /reset-password page reads key + login from searchParams
  • User enters new password → POST /api/auth/reset-password → WP check_password_reset_key() validates, reset_password() saves
  • Success state: inline confirmation + "Go to Login" button. Invalid/missing params: redirect to /forgot-password?invalid=1 with error notice.
Session / Logout
  • Logout: POST /api/auth/logout → WP clears session + cookie
  • requireAuth() server action used on /my-account
  • Unauthenticated → redirect to /login?redirect=/my-account

  • WP endpoint custom/v1/home-page → mmf_build_product_catalog() → for each selected series term, returns one catalog item: { id: term_id, name: term_name, slug: "", sku: "", permalink: "/product-category/{parent}/{child}?series={term_slug}" } — the series name is the link label; clicking navigates to the category page filtered by that series

Cart

Cart State
  • Zustand store: useCartStore
  • Bootstrapped on first load via GET /api/cart
  • WC Store API session managed via Nonce + Cart-Token cookies
  • Guest cart persisted via cart token (no login required)
Operations
  • Add: POST /api/cart/add → wc/store/v1/cart/add-item
  • Remove: DELETE /api/cart/remove → wc/store/v1/cart/remove-item
  • Update qty: POST /api/cart/update → PUT wc/store/v1/cart/update-item
  • Apply coupon: POST /api/cart/coupon/apply
  • Remove coupon: DELETE /api/cart/coupon/remove
  • All mutations return fresh cart state; session cookies are refreshed

Tax Exemption Module

Customer lifecycle: Register (optional cert upload) ──► status: pending │ admin notified by email ▼ Admin APPROVES (email HMAC link / WP admin / user profile) │ ▼ status: approved ──► customer email sent ──► $0 tax at checkout │ (expiry − 30d) ──► expiring soon reminder email ──► banner in My Account │ (past expiry) ──► expired ──► banner + re-upload form │ Customer re-uploads ──► status: pending (cycle repeats) Rejection path: Admin REJECTS → status: rejected → customer email → re-upload form shown
Certificate Upload
  • Allowed: PDF, JPG, JPEG, PNG, DOC, DOCX
  • Max size: 5 MB
  • Server: MIME type validation (not just extension)
  • Stored private — owner-gated download
  • Expiry date required
Admin Approval
  • HMAC-signed email link (14-day expiry)
  • WP Admin → Tax Exemptions list table
  • User profile → approve/reject checkbox
  • One-click AJAX on user list column
Checkout Effect
  • Approved + not expired → TaxJar returns $0
  • Expired/rejected → normal tax applies
  • Checked server-side on every checkout request

Net 30 Terms

How it works
  • COD gateway repurposed as "Net 30 — Purchase Order Terms"
  • Per-customer meta flag: _mmf_net30_enabled
  • Server-side WC filter: gateway only shown to flagged customers
  • At checkout: no card entry required — PO number / notes optional
  • Order created with payment method "Net 30"
Admin controls
  • WP Users list → "Net 30" column → one-click AJAX toggle
  • User profile page → "Net 30 Terms" checkbox
  • WC admin: COD gateway enabled + renamed (one-time setup)

Order Documents

Spec Sheets
  • Available immediately after order is placed (any status)
  • URL stored on order line item meta at purchase time
  • Fallback: reads from product if meta missing
Product Certificates
  • Unlocked when order status = shipped or completed
  • Customer receives email notification on first transition to cert-eligible status
  • "shipped" is a registered WC custom status (not just a string)
  • Email: navy header, gold CTA button, link to My Account

Security Rules

Sessions & Cookies
  • WP session cookies: HttpOnly, Secure, SameSite=Lax
  • WC cart cookies: SameSite=Lax (SameSite=None if 3DS POST-back needed)
  • MMF_PROXY_SECRET must be a long random string (not "1") in wp-config.php + Next.js env
  • Open redirect: only relative paths accepted in ?redirect=
Data Access
  • All order/account endpoints scoped to get_current_user_id()
  • Tax cert files stored private — not public media
  • Card data never touches our servers (PCI SAQ-A via Stripe Elements)
  • No IDOR: ID params validated against current user
Uploads
  • Allowed: PDF, JPG, JPEG, PNG, DOC, DOCX only
  • Server: MIME type check (not just extension)
  • Max 5 MB enforced client + server
  • Files stored outside webroot or private
Input / Admin Actions
  • All admin AJAX actions check check_ajax_referer() + current_user_can()
  • HMAC + 14-day expiry on tax cert approval email links
  • WP REST endpoints sanitize all input (sanitize_text_field, absint, etc.)
  • Next.js proxy caps payload sizes
Headers (Next.js)
  • HSTS, X-Frame-Options: DENY, X-Content-Type-Options: nosniff
  • Referrer-Policy: strict-origin-when-cross-origin
  • Permissions-Policy restricts camera/mic/geolocation
Pre-commit Checklist
  • npx tsc --noEmit — must pass clean
  • npx next lint — zero errors
  • php -l wordpress/inc/*.php — no syntax errors
  • No style= inline CSS introduced
  • No hardcoded prices, currency, or secrets
