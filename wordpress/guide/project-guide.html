Architecture

The browser only ever talks to Next.js. WordPress/WooCommerce is the single source of truth for products, pricing, tax, stock, orders and every setting. Stripe and TaxJar plug into WooCommerce exactly as they would on a classic store.

Browser (React UI, Stripe Elements iframes)
   │  HTTPS only — WP cookies are HttpOnly, never visible to JS
   ▼
Next.js 16  /api/* proxy routes · RSC + ISR pages · security headers
   │  forwards WP session cookies + X-MMF-Proxy shared secret
   ▼
WordPress / WooCommerce (Pantheon)
   ├── WC Store API  → cart · checkout · coupons · shipping · gateways
   ├── custom/v1     → auth · tax-exemption · orders · settings · search
   ├── Stripe Gateway (UPE deferred intent — card charged server-side)
   ├── TaxJar        → rooftop-accurate sales tax (needs full address)
   └── Shippo        → labels + tracking (pending client credentials)

Features Built

Headless Checkout Done

WC Store API checkout: address → live shipping rates → tax → Stripe split card fields (number/expiry/CVC) with on-submit validation → 3DS support → WC-style success page. Guest checkout, login notice, coupons and field visibility all follow WC admin settings.

Tax Exemption Module Done

Certificate upload (register + inline banner), pending/approved/rejected/expiring/expired states, admin one-click email approval (HMAC, 14-day expiry), branded customer emails, daily expiry-reminder cron, TaxJar $0 tax for approved customers. Certificates stored private with owner-gated downloads.

Net 30 Terms Done

COD gateway repurposed per SOW. Per-customer flag — Users-list one-click AJAX toggle + profile checkbox. Server-side gateway filter: unflagged customers never see it, even via API. Checkout skips card entry for Net 30.

My Account (Figma) Done

Sidebar layout: Orders (expandable rows, certificate downloads), Certifications, Spec Sheets, Addresses (WC customer data), Payment Methods, Account Details, Documents, Logout.

Product Catalog + Docs Done

Spec-parts catalog (1,900+ SKUs via CSV import), package pricing tiers, spec sheets + certificates per product, post-purchase document downloads pinned to order line items.

Shippo Integration Pending client

Checkout renders whatever the shipping zones return — temporary flat/free rates now; Shippo live rates drop in with zero frontend changes. Client needs: account, ship-from address, carrier accounts, box sizes, pricing policy.

Key Flows

Order placement

address complete → debounced 800ms → update-customer → rates + tax refresh
card fields valid → createPaymentMethod (browser ↔ Stripe direct)
→ POST /wc/store/v1/checkout {payment_data: wc-stripe keys}
→ [3DS? confirmCardPayment in browser] → order created → /checkout/success
emails: deferred to wp-cron (fixes the old 504 timeout)

Tax exemption lifecycle

upload (register / banner) → status: pending → async admin email
admin APPROVE (email HMAC link · dashboard · profile) → status: approved
→ customer email → checkout tax = $0 while expiry valid
expiry −30d: reminder email → expired: banner + re-upload → cycle repeats

Code conventions

  • No inline CSS (style=) — Tailwind classes only; sole exception: runtime-computed values (progress %, measured offsets)
  • No hardcoded money/currency — format_store_price() (frontend) / mmf_plain_price() (WP) read WC settings
  • Every file has the header comment block (name, description, developer, dates)
  • Pre-commit: npx tsc --noEmit · npx next lint · php -l — all must pass clean

Security audited 2026-07-09

Full two-sided audit (Next.js + WordPress) completed; every High and Medium finding fixed. Details in security-guide.md.

Sessions

  • HttpOnly + Secure + SameSite cookies
  • Proxy shared secret (MMF_PROXY_SECRET)
  • Open-redirect blocked

Data

  • Tax certs private + owner-gated
  • No IDOR — all endpoints scoped to current user
  • Card data never touches our servers (PCI SAQ-A)

Hardening

  • Input caps on every route
  • Nonce + capability on all admin actions
  • HMAC + expiry on email action links
  • HSTS / XFO / nosniff headers
